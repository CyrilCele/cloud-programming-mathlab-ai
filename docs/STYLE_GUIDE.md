# Style Guide

## Terraform

- Two-space indentation
- Descriptive resource names
- Variables documented
- Outputs documented
- Tags applied consistently

---

## Shell Scripts

- Bash only
- Use:

set -euo pipefail

- Quote variables
- Modular functions
- No duplicated logic

---

## Markdown

- ATX headings
- Consistent tables
- Language identifiers on fenced code blocks
- Relative links
- Keep lines readable

---

## Git

Use Conventional Commits.

Examples:

feat:

fix:

docs:

refactor:

test:

ci:

chore:
